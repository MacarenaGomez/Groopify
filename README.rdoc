La aplicación ha requerido dos modelos: Pet y User. Se establece una relación 1:n entre ambas tablas. El peso de la aplicación recae en el controlador de la mascota. Las acciones implementadas en el controlador son las siguientes: new, create, index, edit, update, destroy. No se implementa show (lo he dejado comentado) puesto que se realiza la visualización del detalle de la mascota a través de una llamada ajax que ataca la web como api. El controlador del usuario tiene una única operación para devolver los datos del usuario como api, aunque supongo que se podría haber devuelto en el json de la mascota, pero preferí hacerlo así para no mezcar información de entidades diferentes. La petición ajax se lanza al pinchar sobre el nombre de la mascota. En realidad se realizan dos llamadas ajax que atacan la api. La primera de ellas para obtener los datos de la mascota y la segunda para obtener el nombre del dueño.
